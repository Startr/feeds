Startr/feeds

Self-hosted feed rewriter — own your subscriber URL, rent the audio host.

Open Admin Dashboard

Adding a Feed

In the admin dashboard, open the feeds collection and click New Record. Here's what each field means:

What to rewrite
source_url The RSS feed URL you want to rewrite. This is the original feed from your audio host.
e.g. https://anchor.fm/s/9822e708/podcast/rss
Where to publish
slug A short identifier for your feed — letters, numbers, and hyphens. This single value determines both where the file is saved and its public URL.
e.g. my-show
A slug of my-show produces:
File: pb_public/v1/my-show.xml
URL: https://yourdomain.com/v1/my-show.xml
domain Optional. Override the domain used in the feed's public URL. Leave blank to use the default (set via FEEDS_DOMAIN env var or the PocketBase Application URL in admin settings).
e.g. https://feed.example.com
Your branding
title Your show's name — replaces whatever the upstream feed says.
e.g. The Startr Podcast
website Your show's homepage URL.
e.g. https://yourdomain.com/podcast
cover_image URL to your cover art. Optional — leave blank to keep the upstream image.
e.g. https://yourdomain.com/podcast/cover.jpg
author The show author for Apple Podcasts / iTunes. Optional.
e.g. Jane Doe
owner_email Contact email for Apple Podcasts. Optional.
e.g. podcasts@yourdomain.com
Schedule (optional)
schedule How often to check for upstream changes, as a cron expression. Leave blank for the default (every 15 minutes).
*/15 * * * * — every 15 min (default)
*/5 * * * * — every 5 min
0 * * * * — every hour
0 */6 * * * — every 6 hours

That's it — save the record and the feed will be rewritten on the next cron tick. The first rewrite also runs automatically when the server starts.

Environment Variables (alternative)

For single-feed deploys, you can skip the admin UI and configure via environment variables instead. The server uses these as a fallback when no records exist in the feeds collection.

VariableDescription
FEEDS_SOURCE_URLRSS feed URL to rewrite (required)
FEEDS_SLUGFeed identifier — e.g. my-show (required)
FEEDS_DOMAINYour domain — e.g. https://feed.example.com
FEEDS_TITLEShow title (required)
FEEDS_WEBSITEShow homepage URL (required)
FEEDS_COVER_IMAGECover art URL
FEEDS_ITUNES_AUTHORiTunes author
FEEDS_ITUNES_OWNER_EMAILiTunes owner email
FEEDS_CRONCron schedule (default: */15 * * * *)
