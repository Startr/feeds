# Trademark Notice

The names `Startr/feeds`, `Startr`, `startrcast`, `startr-player`, `startr-subscribe`, and the Startr logo are project marks used to identify this project and its canonical releases. They are reserved separately from the code license.

The code is AGPL-3.0. The project name and marks are not. You can use the code under AGPL; you cannot use the name to imply that your fork is the canonical upstream.

## What you can do

- Use the code under the terms of [AGPL-3.0](./LICENSE).
- Reference the project by name in articles, blog posts, talks, documentation, and issue trackers.
- Say "built with Startr/feeds" or "powered by Startr/feeds" when you deploy an instance.
- Link back to https://github.com/Startr/feeds.
- Contribute code, file issues, open pull requests.

## What you cannot do without permission

- Call your fork `Startr/feeds` or use the `Startr` name in a way that implies it is the canonical project.
- Use `startrcast`, `startr-player`, or `startr-subscribe` as your product name.
- Use the Startr logo in a way that implies endorsement by the upstream maintainers.
- Distribute modified binaries under the same project name.

## Forks

If you fork this project and make substantive changes, please rename the fork. This protects users from confusion about which version is the canonical upstream. It does not prevent you from doing whatever you want with the code itself — the AGPL license covers that. It only covers the name.

Precedent: Mozilla/Firefox uses the same model. The code is open source. The name "Firefox" is a trademark reserved to the project. Debian's `iceweasel` existed because Debian's packaging patched the code but wasn't allowed to use the Firefox mark. That pattern is fine for us too.

## Why this exists

Open source projects with weak trademark policy end up with forks that call themselves the original. Users get confused, security updates go to the wrong place, and upstream loses the ability to vouch for the canonical release. A simple trademark reservation prevents that without limiting anyone's freedom to use the code.
